❓ Hilfe & Einstellungs-Referenz

⚙️ Server Allgemein ▾
NAME
Anzeigename des Servers im Content Manager und in der Serverliste.
PASSWORD
Passwort für Spieler zum Joinen. Leer lassen = kein Passwort.
ADMIN_PASSWORD
Passwort für Admin-Befehle im Spiel (/admin) und für RCON-Verbindungen.
REGISTER_TO_LOBBY
1 = Server erscheint in der öffentlichen Kunos-Serverliste weltweit. 0 = Nur über direkte IP erreichbar.
SUN_ANGLE
Tageszeit als Winkelgrad. 0° = Mitternacht, 48° ≈ Morgen, 180° = Mittag, 320° ≈ Abend.
🔌 Ports & Verbindung ▾
UDP_PORT / TCP_PORT
Hauptport für Spiel-Verbindungen (Standard: 9600). Muss in der Firewall / Router-Portweiterleitung geöffnet sein.
HTTP_PORT
Port für den HTTP-Info-Endpunkt (Standard: 8081). Wird vom Content Manager für Serverdetails abgefragt.
PICKUP_MODE_ENABLED
1 = Spieler wählen ihr Auto selbst beim Joinen. 0 = Zugewiesene Slots (z.B. für Rennevents).
LOOP_MODE
1 = Server startet nach jeder Session automatisch neu (Trackday-Modus). 0 = Server wartet auf manuellen Neustart.
CLIENT_SEND_INTERVAL_HZ
Wie oft pro Sekunde der Server Positionsdaten sendet. Standard: 18 Hz. Höher = flüssiger aber mehr Netzwerklast.
🛡️ Fahrhilfen & Regeln ▾
ABS_ALLOWED
Anti-Blockier-System erlauben. Empfehlung: 1 für öffentliche Server.
TC_ALLOWED
Traktionskontrolle erlauben.
STABILITY_ALLOWED
Elektronisches Stabilitätsprogramm (ESP). Für realistische Server oft deaktiviert.
FUEL_RATE
Kraftstoffverbrauch in %. 100 = realistisch, 0 = unendlich, 200 = doppelt so schnell leer.
DAMAGE_MULTIPLIER
Schadensrate. 0 = kein Schaden, 100 = realistisch, >100 = erhöhter Schaden.
ALLOWED_TYRES_OUT
Wie viele Reifen außerhalb der Strecke erlaubt sind. 2 = Standard, 4 = alle (kein Limit), 0 = strikt.
LEGAL_TYRES
Erlaubte Reifentypen als Kommaliste (z.B. "SV,SM"). Leer = alle Reifen erlaubt.
📅 Sessions ▾
PRACTICE TIME
Dauer der Practice-Session in Minuten. 9999 = unbegrenzt (empfohlen für Trackdays).
IS_OPEN
1 = Spieler können jederzeit joinen. 0 = Kein Beitritt nach Session-Start.
RACE LAPS
Rundenanzahl für das Rennen. 0 = zeitbasiertes Rennen (dann TIME setzen).
RACE_OVER_TIME
Sekunden nach dem Zieleinlauf des ersten Fahrers bis die Session endet. Standard: 180s.
WAIT_TIME
Wartezeit in Sekunden vor Race-Start nach dem Ende des Qualifyings.
🌤️ Wetter & Strecke ▾
GRAPHICS (Wetter)
Setzt das visuelle Wetter-Preset. Presets: 3_clear (sonnig), 2_light_clouds (leicht bewölkt), 7_heavy_clouds (stark bewölkt), 8_drizzle (Nieselregen), 11_practice_storm (Gewitter).
BASE_TEMPERATURE_AMBIENT
Lufttemperatur in °C. Beeinflusst Motortemperatur und Reifenverhalten.
BASE_TEMPERATURE_ROAD
Streckentemperatur in °C. Wärmer = mehr Grip bei Slicks, kälter = weniger.
SESSION_START (Dynamic Track)
Anfangsgripniveau in %. 95 = fast ideal, 80 = grüne Strecke. Steigt durch Runden.
LAP_GAIN
Wie stark der Grip pro Runde zunimmt. 132 = Standard-Nordschleife.
🔧 AssettoServer (extra_cfg.yml) ▾
EnableServerDetails
Muss aktiviert sein damit Beschreibung, Logo und Details im Content Manager erscheinen.
EnableAntiAfk
Spieler die zu lange AFK sind werden automatisch gekickt.
MaxPing
Maximaler Ping in ms bevor ein Spieler automatisch gekickt wird. Standard: 500ms.
EnableWeatherFx
Aktiviert CSP WeatherFX für Regen und flüssige Wetterübergänge. Benötigt CSP 0.1.76+.
MinimumCSPVersion
Minimale CSP-Version zum Joinen (0 = kein Limit). Beispiel: 2144 für CSP 0.1.77.
RconPort
Port für RCON-Verbindungen. Standard: 9700. Wird auch vom Dashboard für Kick/Ban/Chat verwendet.
📜 RCON-Befehle ▾

RCON-Befehle können im Logs-Tab eingegeben werden. Admin-Passwort aus server_cfg.ini wird automatisch verwendet.

/say <text>
Nachricht an alle Spieler senden (erscheint im Spiel-Chat).
/kick_id <id>
Spieler mit Slot-ID kicken (ID sichtbar in Fahrer-Tab).
/ban_id <id>
Spieler bannen und kicken.
/next_session
Zur nächsten Session springen.
/restart_session
Aktuelle Session neu starten.
/set_ballast <id> <kg>
Ballast für einen Spieler live setzen.
